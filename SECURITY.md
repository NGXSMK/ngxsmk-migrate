# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

The `ngxsmk-migrate` team takes security seriously. If you discover a security vulnerability, please do NOT open a public issue. Instead, please report it via the following process:

1. Email your findings to `security@ngxsmk.com`.
2. Include a detailed description of the vulnerability, steps to reproduce, and potential impact.
3. Our team will acknowledge your report within 48 hours and provide a timeline for a fix.

## Security Principles

- **Safe By Default**: All code modifications require automated validation and mandatory backups.
- **Least Privilege**: The toolkit only accesses files within the specified Angular workspace.
- **Transparency**: Every AI-driven change is logged and reported in `ngxsmk-report.md`.

Thank you for helping keep the Angular community safe!
